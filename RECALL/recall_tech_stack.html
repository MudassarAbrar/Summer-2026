AI Layer — 100% Free
Primary AI — Summarization & Categorization
Free
Gemini 2.0 Flash (Google AI Studio)
The best permanently-free AI API available right now. 1,500 requests/day, 1M token context window, multimodal (can read text, images, and YouTube video content). No credit card needed. This is your primary AI for summarizing articles, LinkedIn posts, tweets, and blogs.
1,500 req/day, 15 RPM, 1M token context — permanently free.
Gemini for YouTube
Gemini natively understands YouTube URLs. Pass a YouTube link directly to the Gemini API and ask it to summarize the video — it uses the transcript internally. This solves the long-video problem completely without any extra integration.
Same free tier. No extra setup needed.
Semantic Search (Find by Meaning)
Free via Supabase
pgvector + Supabase Vector Store
Supabase has built-in vector search via pgvector extension. When a link is saved, generate an embedding using Gemini's embedding model (also free) and store it. When the user searches, convert query to embedding and do similarity search. This powers "find that video about cold email" even without exact keywords.
Free within Supabase free tier. Gemini embeddings also free.
Scheduled Jobs (Reminders & Digest)
Free
Supabase pg_cron + Edge Functions
pg_cron is a Postgres extension that runs scheduled SQL. You can set up: a daily job that checks for unactioned items older than X days and sends push notifications, a Sunday job that compiles the weekly digest and sends it. All inside Supabase — no extra service needed.
Free within Supabase.
Backup AI (overflow days)
Free
Groq (Llama 3.3 70B)
If you hit Gemini's daily limit, fall back to Groq. Extremely fast, 14,400 req/day free, OpenAI-compatible API. Same prompts, different provider. Build a simple fallback in your Edge Function.
Technical Feasibility — Honest Assessment
Fully feasible — no issues
Link capture (share sheet + browser extension), AI summarization, categorization, database storage, auth, push notifications, web dashboard, semantic search — all of this is well-documented, free, and achievable solo as a student project. The stack is battle-tested.
Risk 1 — Paywalled content
Some links won't be fetchable. Medium paywalled articles, LinkedIn posts requiring login, private Instagram posts — the app can't read them server-side because it has no session cookies for those platforms.
Fix: Show a clear message — "We couldn't read this content. Add a note to help categorize it." User adds a short note. AI uses title + note + domain name to categorize. Not perfect but practical.
Risk 2 — TikTok / Instagram content
TikTok and Instagram don't expose video transcripts publicly. The app can extract the video title, username, and caption from the page — but not a full transcript like YouTube.
Fix: Same approach as long videos — title + caption + user note. The AI summarizes what it can see. For Phase 2, explore unofficial transcript APIs (e.g. TikTok's caption track if available in the video embed).
Risk 3 — Rate limits at scale
Gemini's free tier is 1,500 req/day. For personal use that's plenty. But when you launch publicly with 100+ users each saving 10+ links/day, you'll hit the ceiling fast.
Fix: Queue system. Don't process links instantly — add them to a queue, process in batches. For personal MVP, no issue. For public launch, this is when you'd add a very small paid Gemini tier (or monetize the app and cover costs). Groq as fallback buys time.
Risk 4 — iOS App Store publishing
Publishing to the Apple App Store costs $99/year. There's no way around this for native iOS distribution.
Fix for now: Use Expo Go for personal testing (free). For Android public launch, Google Play is just $25 one-time. iOS stays internal (TestFlight is free) until you're ready to pay the $99 — or monetize the app first to cover it.
Chrome Web Store — $5 one-time
Publishing the browser extension to Chrome Web Store costs a one-time $5 developer fee. Edge Add-ons store is completely free as an alternative. For zero cost, launch the extension on Edge/Brave first, then pay the $5 for Chrome when you're ready.
Supabase free tier is genuinely enough
500MB database and 50k monthly active users on the free tier. For personal use and early public launch (first few hundred users), you won't need to upgrade. Supabase also has a pause feature — inactive projects stay free indefinitely.
